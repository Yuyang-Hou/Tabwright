The Playwright CLI is excellent for browser tests and scripted automation. Tabwright is designed for a
different completion condition: preserve one successful browser task as a Skill that future agent sessions
can discover, authorize, and run safely.

## Comparison

|  | Playwright CLI | Tabwright |
|---|---|---|
| Primary use | Tests and one-off browser commands | Reusable browser-backed Agent Skills |
| Signed-in Chrome | Depends on connection setup | Per-tab extension control |
| Execution contract | Script and CLI arguments | Strategy, schemas, permissions, auth, side effects |
| Protected request | Script the required page flow | First-class hybrid UI + network workflow |
| Human verification | Implement in the script | Standard resumable `needs_human` checkpoint |
| Distribution | Share code and configuration | Export a portable Agent Skill |
| Raw CDP helpers | Not the main CLI surface | Built-in debugging and inspection helpers |

## Same API, different product boundary

Tabwright uses the same Playwright API you already know, but successful browser work can be promoted into
a durable capability contract and distributed to an Agent Skill manager.

```bash
# Playwright CLI: run a browser command
npx playwright screenshot https://example.com shot.png

# Tabwright: compile and run a reusable workflow
tabwright replay make <replay-id> capture-report --goal "capture the report" --force --json
tabwright capability run capture-report --input-json '{}' --json
```

## Raw CDP access

Tabwright exposes the full Chrome DevTools Protocol through `getCDPSession`. This lets
agents set breakpoints, profile performance, intercept network requests, and live-edit
page scripts through its agent-oriented runtime.

```bash
tabwright -s 1 -e "state.cdp = getCDPSession({ page })"
tabwright -s 1 -e "state.cdp.send('Performance.getMetrics').then(console.log)"
```

## Video recording

Playwright CLI records via tracing (file-based, low FPS). Tabwright uses
`chrome.tabCapture` for native 30-60fps recording that survives page navigation.

```bash
tabwright -s 1 -e "recording.start({ page, outputPath: './demo.mp4' })"
tabwright -s 1 -e "page.click('a'); page.waitForLoadState('domcontentloaded')"
tabwright -s 1 -e "recording.stop({ page })"
```
